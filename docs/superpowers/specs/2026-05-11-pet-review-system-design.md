# 家养宠物评价及攻略系统 — 设计文档

## 概述

一个动物评价及攻略分享网站。用户通过手机号注册，可对动物发表评论、分享攻略。管理员可管理所有用户和内容。全站部署于阿里云单台 ECS，追求低成本与快速上线。

## 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Element Plus | 国内主流 UI 库，开发快 |
| 后端 | Spring Boot 2.7 + MyBatis-Plus | RESTful API |
| 数据库 | MySQL 8.0 | 免费 |
| 缓存 | Redis | 短信验证码 + 登录 Token 缓存 |
| 反向代理 | Nginx | 静态资源 + API 转发 |
| 短信 | 阿里云短信服务 | 0.045 元/条 |
| 部署 | 单台阿里云 ECS（2核4G） | 约 60-80 元/月 |

## 架构（简化版）

```
浏览器 → Nginx → Vue 前端（静态文件）
              → /api/* → Spring Boot → MySQL + Redis
```

全部跑在同一台服务器上。

## 功能范围（速度优先，只保留核心）

### 普通用户
- 手机号 + 验证码注册，设置密码
- 手机号 + 密码登录
- 浏览首页动物列表
- 查看动物详情（介绍 + 评论 + 攻略）
- 发表评论（文字 + 评分）
- 发表攻略（标题 + 正文）
- 个人中心：查看/删除自己的评论和攻略

### 管理员
- 管理后台独立页面
- 用户管理：查看、删除普通用户
- 内容管理：删除任意评论、攻略
- 动物管理：新增、编辑、删除动物

## 数据库表（6 张表）

```sql
-- 用户表
users (id, phone, password_hash, role, created_at)

-- 验证码表
sms_codes (id, phone, code, expires_at, used)

-- 动物表
pets (id, name, category, image_url, description, created_at)

-- 评论表
reviews (id, user_id, pet_id, content, rating, created_at)

-- 攻略表
guides (id, user_id, pet_id, title, content, created_at)

-- 管理员操作日志
admin_logs (id, admin_id, target_type, target_id, action, created_at)
```

## 前端页面（5 个页面）

| 页面 | 路由 | 内容 |
|------|------|------|
| 首页 | `/` | 顶部搜索栏 + 动物卡片网格，每张卡片含图片和名称 |
| 动物详情 | `/pet/:id` | 大图 + 介绍 + 评论区 + 攻略区 |
| 登录注册 | `/login` | 手机号 + 验证码 + 密码表单 |
| 个人中心 | `/user` | 我的评论、我的攻略 |
| 管理后台 | `/admin` | 用户列表、评论列表、攻略列表、动物管理 |

## 动物图片来源

- 优先使用 Unsplash / Pexels 等免费图库的公开 URL
- 备选：下载后本地存储，Nginx 直接提供

## 落地流程（并行推进，时间优先）

### 阶段 1：开发（约 7 天）
1. 搭建项目骨架 + 数据库建表
2. 用户注册/登录 + 短信验证
3. 动物 CRUD + 首页/详情页
4. 评论/攻略接口 + 前端页面
5. 管理后台
6. 数据填充 + 美化收尾

### 阶段 2：服务器 & 域名（与开发并行）
1. 购买阿里云 ECS
2. 购买域名
3. 提交 ICP 备案（等待期间用 IP 测试）

### 阶段 3：部署上线（备案通过后 1 天）
1. 服务器环境安装（Nginx、JDK、MySQL、Redis）
2. 前端打包部署 + 后端部署
3. 域名解析 + Nginx 配置
4. 阿里云短信服务正式开通

## 约束

- 单机部署，无负载均衡，日活几千以内可行
- ICP 备案是外部阻塞点（15-20 工作日），备案通过前用 IP 访问
- 短信验证码依赖阿里云短信服务，需签名+模板审核
- 成本：服务器 ~70 元/月 + 域名 ~60 元/年 + 短信 ~22 元/500条
